Invariants & safety nets in current code

Properties the current pipeline depends on. Re-verify each time you touch the surrounding code — these are the things that fail silently if broken.

AE-01 ID stability across runs Held

Stable slug IDs from (surname, year, title) for papers / references; (surname_norm, given_token) for authors with numeric suffix on genuine collisions. Hash-based or row-index IDs are forbidden — the dedup step assumes the first row in a cluster names the cluster.

src/citegraph/ids.py · src/citegraph/authors.py (_make_author_id)

AE-02 Cache compatibility on schema change Held

Pydantic extra="ignore" drops unknown keys (so old caches with a stored Authors field still load); Authors is a derived @property recomputed from Authors_List. Renaming a schema field breaks LLM contract + on-disk cache simultaneously — bump cache compatibility deliberately.

src/citegraph/schemas.py

AE-03 Per-paper failure isolation Held

Each per-paper extraction body is wrapped in try/except Exception. On failure: row appended to metadata_failures.jsonl / references_failures.jsonl, cache not written, loop continues. Sidecar file is removed when the run ends clean — path.exists() ⇔ failures occurred.

src/citegraph/pipeline.py (extract_paper_metadata / extract_paper_references / _write_failures)

AE-04 LLM truncation safety Held

References stage: response_was_truncated inspects finish_reason (enum or string, both shapes handled) and retries once at 2× output cap, capped at 80k. If JSON still fails to parse, fix_incomplete_json_string is the last-resort repair. Changing the JSON shape of references requires also updating the repair function.

src/citegraph/llm.py · src/citegraph/extract_references.py

AE-05 OpenAlex IDs preserved through dedup Held

OpenAlex_Authors carries OpenAlex author.id + ORCID, positionally aligned with Authors_List, from the enrichment stage into enriched_references.csv. The author stage treats these as ground truth for identity (same id ⇒ same cluster; different ids ⇒ different clusters). Dropping the column degrades author normalization to string-only evidence.

src/citegraph/enrich.py (_normalize_record) · src/citegraph/authors.py

AE-06 Bibliography slice never drops the paper Held

slice_to_references_section uses a strict ##-style header regex but falls back to the full markdown when no header matches, so a paper with an oddly-styled bibliography section still gets processed end-to-end (just with more input tokens). The inline-citation sanity check is gated on successful slicing to avoid false-positive warnings on unsliced papers.

src/citegraph/extract_references.py

Conventions worth surfacing

Code

  • Schemas are DTOs and LLM response schemas. PaperMetadata / Reference are passed to Gemini as response_schema and used as the on-disk cache format. Renaming fields breaks both contracts at once.
  • Per-stage I/O lives in OutLayout. Don't hardcode paths under out_dir; add to src/citegraph/io.py.
  • Provider isolation. All Gemini-specific code lives in src/citegraph/llm.py. Other modules speak Pydantic, not Gemini types.
  • Lazy imports for heavy deps. docling and networkx are imported inside their functions so import citegraph stays cheap.
  • Tests never hit Gemini. Use a _FakeClient subclass that sniffs the prompt to distinguish metadata from references — update the sniff if you rewrite prompts.
  • research/ is not in the package. Excluded from CI, lint-loosened. Don't import from it in src/.

Pipeline shape

  • Every stage is checkpointed. Re-runs skip work that's already done; an interrupted LLM pass doesn't waste API calls.
  • Per-stage CLI mirrors Pipeline stage methods. Both surfaces accept input explicitly or read prior artifacts from out_dir; missing upstream raises StageNotReadyError with a hint at the prior step.
  • Sidecar files: absent = clean. Six files (conversion_warnings.json, source_duplicates.json, papers_no_references.json, two failure logs, author_review.json) only exist when their condition occurred.
  • Counts surface in run_summary.json. Six matching count keys (n_conversion_warnings, n_source_duplicates, …); CLIs emit a yellow line when any are non-empty.
  • User-facing knobs live in two places. Pipeline(...) kwargs and the matching CLI flags. Keep them in sync when adding one.
